Please note that the error means, that it cannot load the shared library 
of R (libR.so) itself. The JRI library (libjri.so) seems to be resolved 
and loaded correctly. So the rJava configuration is probably correct.
You should check if the file libR.so exists (usually at 
R_HOME/lib/libR.so). If not, you have to recompile R with the shared 
library option (--enable-R-shlib). If the file exists, please send me 
the real complete path.

>>>> I have just been through the process (with Stephan's help)
>>>> to get an R session, running remotely from a server, into eclipse on 
>>>> my pc.
>>>> so here are a few comments just fyi.
>>>> You should still follow the installation instructions etc,
>>>> in preferences to these personal comments,
>>>> as my setup will probably differ to yours.
>>>>
>>>> I am running eclipse on Vista and the remote server is running Linux 
>>>> Redhat.
>>>>
>>>> Where ever you place the script RJServer/startup.sh (and all the other
>>>> files in the tar file)
>>>> on the remote machine, that file location has to entered on your 
>>>> local machine
>>>> (in eclipse - run - run configuration - main tab - Remote command 
>>>> entry).
>>>> e.g. ~/.RJServer/startup.sh "ldc-sv-otick"
>>>>  where ~ on the remote server means /opt/ontick (for me)
>>>> so the script (and the other files) should be located at
>>>> /opt/ontick/.RJServer/startup.sh
>>>> Dont forget to make the script executable
>>>> via something like chmod u+x /opt/ontick/.RJServer/startup.sh.
>>>>
>>>> Edit the script /opt/ontick/.RJServer/startup.sh
>>>> to add user specific details.
>>>> In my case, I specified the following details:
>>>>
>>>> # local repository of packages including rJava.
>>>> R_LIBS=/opt/ontick/R/library
>>>> R_HOME=/usr/lib64/R
>>>> JAVA_HOME=/usr/java/jdk1.6.0_12/
>>>> HOSTADDRESS=99.88.180.66
>>>>
>>>> Run rmi registry and then manually run the starup script
>>>> on the remote machine to test that it working.
>>>>
>>>> nohup $JAVA_HOME/rmiregistry "ldc-sv-otick" &
>>>>
>>>> Perhaps use, "netstat -ntlp" and look for port 1099,
>>>> the default, for the rmi registry entry.
>>>> "ps -edf | grep -i rmiregistry"  to search for the process.
>>>>
>>>> Test out the script:
>>>>
>>>> ~/.RJServer/startup.sh "ldc-sv-otick" -debug
>>>>
>>>> If all is well, run something like
>>>> nohup ~/.RJServer/startup.sh "ldc-sv-otick" &
>>>>
>>>> On my pc, in Eclipse - run - run configuration - main tab - Remote
>>>> command entry:
>>>>
>>>> Address: //99.88.180.66/ldc-sv-otick (this uses the default port 1099).
>>>> SSH port: 22 (the default)
>>>> Username: (to login to remote server)
>>>> Remote command: ~/.RJServer/startup.sh "ldc-sv-otick"
>>>>
>>>> Hope this helps a bit.
